# EchoWord - 产品需求文档 (PRD)

## 1. 产品概述

**产品名称：** EchoWord

**定位：** 轻量级跨平台划词翻译工具，核心差异化功能为音标中文谐音标注，帮助不熟悉国际音标的用户快速理解英语发音。

**技术栈：** Tauri (Rust + Web 前端)

**目标平台：** macOS、Windows（P0），Linux（P2，Tauri 天然支持，后续迭代补全）

**当前语言支持：** 英 → 中（架构预留多语言扩展能力）

---

## 2. 核心功能

### 2.1 翻译触发方式

| 触发方式 | 说明 | 优先级 |
|---------|------|--------|
| 全局快捷键 | 选中文本后按快捷键触发翻译，读取系统剪贴板 | P0 |
| 鼠标选中自动弹窗 | 在任意应用中选中文本后自动弹出翻译气泡（需辅助功能权限） | P0 |
| 输入翻译 | 通过快捷键唤出输入框，手动输入文字查询 | P0 |
| 剪贴板监听 | 开启后，复制任意文本自动触发翻译，适用于 PDF 阅读、终端等不便划词的场景 | P1 |

**适用场景：** 浏览器、IDE（VSCode 等）、微信、PDF 阅读器、终端、任意桌面应用。

### 2.2 翻译能力

#### 2.2.1 智能模式判断

- 单个单词（无空格） → **单词模式**：显示释义 + 音标 + 谐音
- 多个单词 → **句子模式**：显示整句翻译
- 弹窗内提供手动切换按钮，允许在两种模式间切换

#### 2.2.2 单词翻译

- **数据来源：** 离线词典（优先）或有道词典 API
- **展示内容：**
  - 英文单词
  - 中文释义
  - 国际音标（美式/英式）
  - 音标中文谐音标注（混合方案 B）

#### 2.2.3 句子翻译

- **数据来源：** 在线翻译 API（DeepL/腾讯/百度/有道，用户可配置）
- **展示内容：**
  - 原文
  - 中文翻译

#### 2.2.4 多引擎对照翻译

- 支持同时展示多个翻译引擎的结果，方便用户对比择优
- 用户可在设置中勾选启用的翻译引擎（最多同时展示 3 个）
- 弹窗内以标签页或堆叠形式展示各引擎结果
- 优先级：P1

#### 2.2.5 开发者命名拆分

- 自动识别 camelCase（`getUserName`）和 snake_case（`get_user_name`）命名
- 拆分为独立单词后进行翻译（`get User Name` → 获取用户名）
- 适用于 IDE 中选中变量名/函数名翻译的场景
- 优先级：P1

### 2.3 音标中文谐音标注（核心差异化功能）

采用 **混合标注方案**：

- 能对应的音标：用最近似的中文字标注
- 无法对应的音标：保留原始音标符号 + 发音提示文字

示例：

```
ephemeral /ɪˈfemərəl/
谐音：一 · 飞 · 摸 · 若 · 了

think /θɪŋk/
谐音：[θ咬舌送气] · 英 · 克
```

## 3. 翻译源管理

### 3.1 离线词典

| 版本 | 词条数 | 体积 | 说明 |
|------|--------|------|------|
| 核心版 | 5 万 | ~50MB | 内置，覆盖日常常用词 |
| 精简版 | 20 万 | ~200MB | 可选下载 |
| 完整版 | 77 万 | ~800MB | 可选下载 |

- 基于 ECDICT 开源词典
- 默认内置核心版，精简版和完整版通过应用内词典管理页面一键下载
- 离线词典查询失败时，自动降级到在线 API

### 3.2 在线翻译 API

| API | 音标支持 | 免费额度 | 用途 |
|-----|---------|---------|------|
| DeepL | ❌ | 50 万字符/月（Free 计划） | 句子翻译（默认，翻译质量最佳） |
| 腾讯翻译君 | ❌ | 500 万字符/月 | 句子翻译（备选） |
| 百度翻译 | ❌ | 标准版免费（QPS=1） | 句子翻译（备选） |
| 有道词典 | ✅ | 体验金 50 元 | 单词翻译（备选） |

### 3.3 AI 翻译源

| API | 特点 | 联网要求 | 用途 |
|-----|------|---------|------|
| OpenAI（GPT） | 支持上下文理解、语气润色 | 在线 | 句子/段落翻译，可自定义 prompt |
| Ollama 本地模型 | 完全离线，隐私友好 | 离线 | 隐私敏感场景，本地部署大模型翻译 |

- 用户自行填写 API Key 或配置 Ollama 端点地址
- AI 翻译源采用流式响应，翻译结果逐字输出
- 架构预留通用 LLM 接口，便于后续接入更多模型（Gemini、Claude 等）
- 优先级：P1（OpenAI），P2（Ollama）

### 3.4 翻译缓存与限频策略

#### 缓存策略

- 本地 LRU 缓存最近 500 条翻译结果（内存缓存，重启清空）
- 缓存 key 为 `翻译源 + 源语言 + 目标语言 + 原文`
- 命中缓存时跳过 API 请求，响应 < 5ms
- 离线词典查询结果不走缓存（已经足够快）

#### 请求防抖

- 划词翻译：文本选中后延迟 300ms 触发，避免选中过程中的多次触发
- 输入翻译：停止输入 500ms 后自动翻译（可选开启）
- 剪贴板监听：防止同一文本重复翻译（对比上次翻译内容）

#### API 降级链

- 默认降级链：DeepL → 腾讯翻译君 → 百度翻译
- 当前翻译源返回错误（超时 / 限频 / 鉴权失败）时，自动尝试下一个翻译源
- 降级时在弹窗内显示小提示（如"已切换至备用翻译源"）
- 用户可在设置中自定义降级顺序

### 3.5 配置能力

- 用户可在设置页面切换翻译源
- 支持离线/在线模式切换
- 用户自行填写 API Key
- 支持自定义 API 降级链顺序

## 4. 弹窗交互

### 4.1 气泡弹窗

- **位置：** 跟随鼠标，出现在选中文字附近
- **窗口位置记忆：** 用户手动拖拽弹窗后，后续弹窗优先出现在上次拖拽位置（P2）
- **关闭方式：**
  - 点击弹窗外部区域
  - 按 Esc 键
- **不做超时自动消失**（用户需要时间查看谐音、点收藏、点复制）

### 4.2 弹窗内容（单词模式）

```
┌─────────────────────────────────┐
│ ephemeral            🔊 ⭐ 📋  │
│ /ɪˈfemərəl/                    │
│ 谐音：一 · 飞 · 摸 · 若 · 了    │
│─────────────────────────────────│
│ adj. 短暂的；转瞬即逝的          │
│                                 │
│ [单词模式 | 句子模式]            │
└─────────────────────────────────┘
```

### 4.3 弹窗内容（句子模式）

```
┌─────────────────────────────────┐
│ I love coding            🔊 📋 │
│─────────────────────────────────│
│ 我喜欢编程                      │
│                                 │
│ [单词模式 | 句子模式]            │
└─────────────────────────────────┘
```

### 4.4 键盘导航

- 弹窗内支持键盘操作，提升效率用户体验：
  - `Tab` / `Shift+Tab`：在操作按钮间切换焦点（朗读、收藏、复制）
  - `Enter`：执行当前焦点按钮的操作
  - `Esc`：关闭弹窗
  - `Ctrl/Cmd + C`：快速复制翻译结果
- 优先级：P1

### 5.2 收藏列表

- 查看所有收藏的单词
- 支持搜索（按单词、释义搜索）
- 优先级：P1

### 5.3 导出

- 支持导出为 CSV、JSON、Anki 卡片格式
- 优先级：P2

## 8. 数据存储与同步

### 8.1 本地存储

- 使用 SQLite 存储所有数据（收藏、历史、设置）
- 优先级：P0

### 8.2 数据同步

- 应用数据目录支持用户自定义路径
- 将数据目录设置到坚果云同步文件夹，即可实现多设备同步
- 不需要应用内实现同步逻辑，利用坚果云客户端的文件同步能力
- 优先级：P1

### 8.3 同步冲突处理

- SQLite 数据库在多设备通过坚果云同步时，可能产生写入冲突
- 处理策略：
  - 应用启动时检测数据库锁定状态，若被占用则等待重试（最多 3 次，每次间隔 1s）
  - 坚果云产生冲突文件时（如 `data.db (NSConflict xxx)`），在设置页面提示用户手动选择保留版本
  - 写入操作采用 WAL 模式，减少锁冲突概率
- 优先级：P1

### 8.4 数据库 Migration

- 使用版本号管理数据库 schema（在 SQLite 中维护 `schema_version` 表）
- 每次应用启动时检查当前 schema 版本，自动执行增量迁移脚本
- 迁移脚本在事务中执行，失败时回滚并提示用户
- 迁移前自动备份数据库文件（`data.db.bak.{version}`）
- 优先级：P0

## 9. 系统集成

### 9.1 系统托盘

- 关闭窗口 → 最小化到系统托盘（非退出）
- 托盘图标右键菜单：显示主窗口、隐私模式开关、设置、退出
- 优先级：P0

### 9.2 开机自启

- 可选开机自动启动
- 优先级：P2

### 9.3 外部调用 API

- 提供本地 HTTP 接口（默认 `127.0.0.1:16888`），供其他应用集成调用
- 端口号可在设置中修改
- API 列表：

| 接口 | 方式 | 说明 |
|------|------|------|
| `/translate` | POST | 翻译指定文本（body 为待翻译文本） |
| `/selection_translate` | GET | 触发划词翻译 |
| `/input_translate` | GET | 唤出输入翻译窗口 |

- 使用场景：
  - macOS 配合 PopClip / Alfred / Raycast 扩展调用
  - Windows 配合 SnipDo / uTools 等工具调用
  - 脚本 / 自动化工作流调用
- 优先级：P1

### 9.4 自动更新

- 利用 Tauri 内置 updater 机制，启动时检查 GitHub Release 新版本
- 检测到新版本后弹窗提示用户，用户确认后自动下载安装
- 支持跳过当前版本提醒
- 可在设置中关闭自动检查更新
- 优先级：P1

## 12. 首次启动引导

### 12.1 引导流程

首次打开应用时，展示分步引导页面：

1. **欢迎页** — 简介产品核心功能（音标谐音标注）
2. **权限申请** — 引导用户授予辅助功能权限（macOS Accessibility）
   - 说明为什么需要此权限（鼠标选中自动弹窗翻译）
   - 提供「前往系统设置」按钮，一键跳转权限页面
   - 权限授予后自动检测并进入下一步
   - 若用户跳过，划词翻译功能不可用，但快捷键翻译和输入翻译正常工作
3. **翻译源配置** — 选择默认翻译源，填写 API Key（可跳过，使用离线词典）
4. **快捷键展示** — 展示核心快捷键（划词翻译、输入翻译），支持自定义
5. **完成** — 引导结束，进入主界面

### 12.2 要求

- 引导流程可跳过，不强制完成
- 引导状态持久化，完成后不再弹出
- 在设置页面提供「重新引导」入口
- 优先级：P0

## 14. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 离线词典查询 < 50ms，弹窗响应 < 200ms，缓存命中响应 < 5ms |
| 体积 | 安装包 < 30MB（不含词典扩展包） |
| 内存 | 常驻内存 < 80MB |
| 架构 | 预留多语言扩展能力（当前仅英→中），翻译源接口抽象化便于扩展 |
| 安全 | API Key 本地加密存储，不上传；外部调用 API 仅监听 localhost |
| 稳定性 | 数据库 Migration 失败自动回滚，翻译源故障自动降级 |
| 可观测 | P2 阶段接入崩溃上报（Sentry 或类似方案），收集未捕获异常 |
