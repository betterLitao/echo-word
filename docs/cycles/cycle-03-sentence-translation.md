# 开发周期 03：句子翻译与触发链路

## 周期目标

在单词翻译闭环之上，完成句子翻译主链路、在线翻译源调度、缓存/降级、全局快捷键和输入翻译，让 EchoWord 进入真正可日常使用的阶段。

## 本周期范围

### 必做功能

- 实现 `DeepL`、`腾讯翻译`、`百度翻译` 的统一接入。
- 完成自动模式下的“单词 / 句子”判断与手动模式切换。
- 实现翻译缓存、请求防抖和 API 降级链。
- 完成全局快捷键触发翻译和输入翻译入口。
- 完成句子翻译结果 UI、加载态和降级提示。
- 完成翻译源设置页的最小闭环，包括 API Key 存储。

### 目标模块

- API：`src-tauri/src/api/deepl.rs`、`src-tauri/src/api/tencent.rs`、`src-tauri/src/api/baidu.rs`
- 服务：`src-tauri/src/services/translator.rs`、`src-tauri/src/services/cache.rs`
- 工具：`src-tauri/src/utils/crypto.rs`
- 命令：`src-tauri/src/commands/translate.rs`、`src-tauri/src/commands/settings.rs`
- 前端：`src/components/popup/SentenceResult.tsx`、`src/components/settings/TranslationTab.tsx`
- 状态：`src/stores/translationStore.ts`、`src/stores/settingsStore.ts`

## 开发任务

### Rust 后端

- 完成 `TranslationProvider` 抽象和 Provider Registry。
- 在 `translator.rs` 中接入模式判断、缓存读取、Provider 调度和降级逻辑。
- 将默认降级链实现为 `DeepL -> 腾讯 -> 百度`，并允许从设置页读取顺序。
- 对 API Key 做本地加密存储，避免明文写入 `settings` 表。
- 注册全局快捷键，支持“读取剪贴板后翻译”和“唤出输入翻译”。

### 前端 UI

- 完成句子结果组件，展示原文、译文、当前翻译源和降级提示。
- 在弹窗中打通自动模式与手动模式切换。
- 完成设置页中的翻译源配置、API Key 输入、降级链配置基础表单。
- 为输入翻译提供最小可用入口，可以是独立对话框，也可以复用主窗口特定视图。

### 系统集成

- 快捷键触发时从剪贴板读取文本，进入统一翻译流程。
- 输入翻译支持 500ms 防抖，避免每次键入都立即请求在线服务。
- 对在线翻译错误做统一提示，区分限频、鉴权失败、网络不可达三类场景。

### 验证清单

- 无空格文本默认走单词逻辑，有空格文本默认走句子逻辑。
- 全局快捷键可在其他应用中触发翻译。
- 输入翻译可独立触发且不会阻塞主窗口。
- 重复查询同一段文本时能命中缓存。
- 首选翻译源失败后可自动切换到备用源。

## 交付物

- 可用的在线句子翻译能力。
- 可配置的翻译源与 API Key 存储。
- 快捷键 + 输入翻译双触发链路。
- 缓存、降级、防抖的主调度系统。

## 验收标准

- 用户能够通过全局快捷键完成一次完整翻译闭环。
- 用户能够通过输入翻译入口完成一次完整翻译闭环。
- API Key 在本地存储时不以明文出现。
- 降级链触发后，界面会给出明确提示。

## 非本周期范围

- 鼠标选中自动弹窗
- 首次启动完整引导
- 历史记录、朗读、主题、剪贴板监听
- 词典管理、HTTP API、多引擎对照

## 风险与依赖

- 在线翻译源的鉴权规则不同，需要先定义统一错误码映射。
- 全局快捷键在不同系统上可能存在冲突，需要预留冲突提示能力。
- 剪贴板读取策略要避免误触发历史文本，防止翻译结果与用户预期不一致。

## 建议 PR 拆分

1. Provider 抽象与在线翻译实现
2. 缓存、降级链、统一翻译调度
3. 全局快捷键与输入翻译链路
4. 设置页翻译源配置与 API Key 存储
