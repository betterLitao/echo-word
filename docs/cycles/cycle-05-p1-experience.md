# 开发周期 05：P1 体验增强

## 周期目标

在稳定的 `P0` 基础上，补齐高频使用体验功能，让 EchoWord 从“能用”进入“好用、愿意长期打开”的阶段。

## 本周期范围

### 必做功能

- 发音朗读
- 翻译结果一键复制
- 收藏列表查看 / 搜索
- 历史记录查看 / 搜索
- 快捷键自定义与冲突提示
- 深色 / 浅色 / 跟随系统主题
- 剪贴板监听模式
- 开发者命名拆分（camelCase / snake_case / kebab-case）
- 弹窗键盘导航

### 目标模块

- 服务：`src-tauri/src/services/tts.rs`、`src-tauri/src/services/dev_name.rs`
- 命令：`src-tauri/src/commands/history.rs`、`src-tauri/src/commands/favorite.rs`
- 前端页面：`src/components/settings/ShortcutTab.tsx`、`src/components/settings/GeneralTab.tsx`
- 前端列表：`src/pages/favorites`、`src/pages/history`（若后续按页面目录组织）
- Hooks：`src/hooks/useTheme.ts`

## 开发任务

### 用户体验能力

- 接入 TTS 服务，为单词模式提供发音按钮。
- 在弹窗中提供复制原文/译文能力，并给出复制成功反馈。
- 为收藏和历史增加查询、分页或懒加载能力，避免数据增长后页面卡顿。
- 为弹窗补充键盘导航能力，支持切换模式、收藏、复制、关闭。

### 设置与个性化

- 在设置页支持自定义快捷键，并处理旧快捷键注销、新快捷键注册和冲突提示。
- 实现主题切换，并保证主窗口与弹窗窗口主题状态一致。
- 实现剪贴板监听开关与去重逻辑，避免同一文本重复触发。

### 开发者场景增强

- 对 camelCase、snake_case、kebab-case 文本进行拆分后再翻译。
- 拆分后的结果优先走离线词典，再按句子方式聚合输出，保证 IDE 场景可用。

### 验证清单

- 单词结果可播放发音，失败时有降级提示。
- 收藏列表和历史列表可检索并正确分页。
- 修改快捷键后，无需重启应用即可生效。
- 主题切换后，主窗口与弹窗样式保持一致。
- 开发者命名如 `getUserName`、`get_user_name` 能获得合理翻译结果。

## 交付物

- 完整的 `P1` 高频体验能力集合。
- 可查询的收藏与历史视图。
- 更适合开发者和重度用户的交互体验。

## 验收标准

- 用户不依赖外部工具即可完成“查询、朗读、复制、收藏、回看历史”的日常流程。
- 快捷键、主题、剪贴板监听等设置修改后能即时生效。
- 命名拆分不会破坏普通单词或句子翻译路径。

## 非本周期范围

- 词典下载管理
- 数据目录切换与坚果云同步冲突处理
- OpenAI / 有道 / 多引擎对照
- HTTP API 与自动更新
- P2 项目级能力

## 风险与依赖

- TTS 在不同平台的表现可能不一致，需要尽早确定实现方案或能力边界。
- 快捷键自定义与系统冲突检测，容易出现“保存成功但实际不可用”的问题。
- 历史与收藏列表一旦上线，就需要关注分页、索引和数据量增长后的查询性能。

## 建议 PR 拆分

1. TTS、复制、弹窗键盘导航
2. 收藏列表与历史记录页面
3. 快捷键自定义与主题系统
4. 剪贴板监听与开发者命名拆分
