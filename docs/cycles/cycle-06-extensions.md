# 开发周期 06：扩展集成与 P2 储备

## 周期目标

完成 P1 中偏系统和集成侧的能力，同时为 P2 的截图翻译、Ollama、本地隐私和跨平台扩展建立技术储备，使 EchoWord 具备持续演进空间。

## 本周期范围

### 主交付（P1 收尾）

- 数据目录可配置
- 坚果云同步冲突检测与提示
- 词典管理（核心 / 精简 / 完整版下载）
- 有道词典 API
- OpenAI 翻译源
- 多引擎对照翻译
- 外部调用 HTTP API
- 自动更新
- 网络代理配置

### 预研与储备（P2 输入）

- 开机自启
- Ollama 本地模型翻译
- 截图 OCR 翻译
- 弹窗位置记忆
- 隐私模式
- 界面语言切换（i18n）
- Linux 平台适配
- 崩溃上报

### 目标模块

- 命令：`src-tauri/src/commands/dictionary.rs`、`src-tauri/src/commands/settings.rs`
- API：`src-tauri/src/api/youdao.rs`、`src-tauri/src/api/openai.rs`
- 服务：`src-tauri/src/http_server/*`
- 工具：`src-tauri/src/utils/platform.rs`
- 前端：`src/components/settings/DictionaryTab.tsx`、`src/components/settings/TranslationTab.tsx`

## 开发任务

### 扩展翻译能力

- 接入有道词典 API，补强在线单词查询兜底能力。
- 接入 OpenAI 翻译源，支持流式展示并为多引擎对照预留统一输出结构。
- 在弹窗内支持最多 3 个引擎的对照显示，明确主结果与辅助结果的层级关系。

### 系统与数据能力

- 允许用户切换数据目录，并在切换后安全迁移数据库与配置。
- 扫描坚果云冲突文件并在设置页给予提示与处理入口。
- 完成扩展词典下载、进度展示、版本切换和失败恢复。
- 接入代理配置，统一传递给在线翻译与更新检查链路。

### 平台集成能力

- 启动本地 `HTTP API` 服务，支持外部脚本调用翻译能力。
- 接入自动更新并完成最小可用的版本检查、提醒、下载流程。
- 评估并沉淀开机自启、Ollama、OCR、隐私模式等 `P2` 的技术方案、风险和外部依赖。

### 验证清单

- 切换数据目录后，已有数据可保留且应用能正常启动。
- 下载扩展词典时有进度反馈，中断后能安全恢复或重试。
- 多引擎对照模式下，结果展示不遮挡主流程且性能可接受。
- 本地 HTTP API 仅监听 `127.0.0.1`。
- 更新检查失败、代理错误、OpenAI 流式中断等异常路径都有可见反馈。

## 交付物

- `P1` 中偏平台和集成侧能力的完整实现。
- 一份可执行的 `P2` 技术预研清单与优先顺序。
- 面向生态扩展的翻译能力接口和系统接口。

## 验收标准

- 用户可管理词典、切换数据目录、使用 OpenAI/有道等扩展翻译源。
- 外部工具能够通过本地 HTTP API 调用 EchoWord 翻译能力。
- 自动更新、代理配置和同步冲突检测在主流程中不会破坏稳定性。
- `P2` 项目的实现顺序、风险和前置依赖清晰可执行。

## 非本周期范围

- 完整实现全部 P2 功能
- Linux 正式发布支持
- OCR 模型或本地大模型的生产级交付

## 风险与依赖

- 词典下载、自动更新和 OpenAI 流式能力都依赖稳定网络，需要统一错误处理模型。
- 数据目录迁移一旦出现路径或锁文件问题，容易造成用户数据风险，必须先做备份策略。
- 多引擎对照会放大 API 成本与弹窗复杂度，需要严格限制并发数和 UI 密度。

## 建议 PR 拆分

1. 数据目录、同步冲突、词典管理
2. 有道 / OpenAI / 多引擎对照
3. HTTP API、代理配置、自动更新
4. P2 技术预研文档与原型验证
